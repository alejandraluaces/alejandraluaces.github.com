Jekyll Woes

April 05, 2013

For this apprenticeship, I was tasked with setting up my blog using Jekyll and Github Pages. Though I barely managed and had lots of help, I have been able to both maintain it and make massive changes. Like anything, with exposure and a "dive in head first" mentality, learning is somewhat easy. Google searches also help.

Lately, I have been issues with my blog. First, the posts weren't showing up in the right "sections". Next, the images stopped working. Then, the layout.html/styles.css files weren't working. What gives? First, know a couple things about the way my blog is set up.

The way my blog works is that I have two blogs within one. Confusing? Sure. But it works.

|--_config.yml
|--_includes
|--_layouts
|  |--default.html
|  |--post.html
|--blog
|  |--index.html
|  |--_posts
|  |  |--2012-04-06-hello-world.md
|--projects
|  |--index.html
|  |--_posts
|--index.html

Each section should have its own folder, wherein you'll have an index.html, a _posts folder, and perhaps some other folders (although not necessary): css, images, etc. You'll have two different html files in the layouts folder, and you'll direct to them from the yaml front matter included in all of the posts.

Yaml is touchy. I learned this the hard way. My posts were in their respective sections (the _posts folder of each "separate" blog) and I was updating everything just as I should. So, what was the problem?

Punctuation. Punctuation was all the problem. This is something I never would have been able to figure out myself, because I do not think like a programmer. I am not hard wired to question the punctuation in anything, where it's placed, or what it could mean. All I know is that gramatically, it makes sense, and so I'm using it.

At the top of every blog post you'll put in the "_posts" folder, you'll have some Yaml that will tell the post how it will be laid out, where it is going on the page, and what the title and date are. These are all user-specified. If there is any punctuation in these, it will mess up. I had a post in my blog/_posts/ folder whose title contained a colon. That little colon completely derailed my blog.

So, how did I find the error?

My mentor Justin showed me how to run jekyll --no-auto --server, which specifically tells you where your errors may be, the file name/location, and line number. It'll also let you know which character is throwing a wrench into the delicate innerworkings of your site. Pretty sweet.

Erasing the _site directory is another thing that you might want to do as well if you're encountering problems like this. That'll ensure that you're working from a clean slate if you have any corrupt files, and will help to avoid having to make changes to a bunch of files.
