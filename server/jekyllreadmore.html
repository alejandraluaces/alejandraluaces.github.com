Jekyll Read More Functionality

May 22, 2013

Jekyll was completely new to me when I started this apprenticeship, and I find that I still struggle with it sometimes; there are so many possibilities that is met with a startling lack of documentation or resources for people just starting out. Perhaps I can start to get my list of tips and tricks together to make a little more beginner-friendly blog post. But that's later.

So, ever since my user testing project where I first engaged in user testing my blog layout, I have been wanting to add a "Read More.." link to the projects page of my blog so that only an image would appear on the "front", and you could have the option of clicking to read more about my experience in making the project. However, the documentation available online is sparse and I really just wanted to be able to figure it out myself. I pushed this to the back burner while I dealt with more high-priority things, but when I saw Meagan's new blog layout, I saw she had added "Read more.." links to hers!

So, the way she did it was by replacing {{ post.content }} with {{ post.excerpt }} . This was supposed to tell Jekyll to create an excerpt from the available text. Of course, abiding by Murphy's Law, this method did not work for me, so I had to hop on the dreaded Google and do some more research.

I did find some more helpful articles, one of which really helped to explain this to me in a way I could understand. It was important to find something that I could really understand and be able to implement without it being a huge headache for my non-programmery mind. So, this is the solution that worked for me: on my index pages, instead of putting {{ post.content }}, I replaced it with {{ post.content | split: '<!-- more -->' | first }}, making sure to put a link beneath it that'll navigate to the permalink where the user will be able to read the rest of the article. This lets the index pages split your articles wherever the <!-- more --> comment appears.
